# WatchParty Live — React Native App

A React Native mobile app for watching streams and podcasts together with friends.
Bare RN (no Expo). Works on iOS and Android.

> **Stack**: React Native 0.81 · React 19 · TypeScript · React Navigation v7 · `react-native-linear-gradient` · `react-native-vector-icons` · `react-native-image-picker`

---

## 1. Prerequisites

| Tool         | Version              |
| ------------ | -------------------- |
| Xcode        | 26                   |
| iOS Sim      | iOS 17+              |
| React Native | 0.81+                |
| React        | 19                   |
| Node.js      | 22 LTS               |
| CocoaPods    | latest               |
| Java         | 17 (JDK Temurin/Zulu)|
| Android SDK  | 35 / 36              |
| Ruby         | 3.2+ (for CocoaPods) |
| Watchman     | latest (mac)         |

Install on macOS:

```bash
# Node 22
brew install node@22
nvm install 22 && nvm use 22

# Watchman
brew install watchman

# Java 17 (Temurin)
brew install --cask temurin@17

# CocoaPods
sudo gem install cocoapods
```

Set `JAVA_HOME` (zsh):

```bash
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
source ~/.zshrc
```

For Android, install via Android Studio:
- Android SDK Platform 35 (or 36)
- Android SDK Build-Tools 35.0.0
- Android Emulator + a Pixel image (API 34/35)
- NDK 26.x

Add to `~/.zshrc`:

```bash
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/platform-tools
```

## 2. Initial setup (first time)

The `ios/` and `android/` native folders are NOT included in this ZIP — they're huge and platform/version-specific. We'll generate them with the RN community CLI so you get folders that match your exact Xcode/Gradle versions.

From the project root:

```bash
# 1. Install JS deps
npm install

# 2. Generate native folders (uses our package.json, app.json, index.js)
#    This creates ios/ and android/ from the latest RN 0.81 template.
npx @react-native-community/cli@latest init WatchPartyLive \
    --version 0.81 \
    --skip-install \
    --directory _tmp_template

# 3. Move generated native folders into our project
mv _tmp_template/ios ./ios
mv _tmp_template/android ./android
rm -rf _tmp_template
```

Then in `ios/Podfile`, ensure platform line reads:

```ruby
platform :ios, '15.1'
```

Now install pods + fonts:

```bash
# 4. iOS pods
cd ios && pod install && cd ..

# 5. Drop font files into src/assets/fonts/ then link them:
npx react-native-asset
```

## 3. Fonts

Add to `src/assets/fonts/`:

- `SchibstedGrotesk[wght].ttf`
- `SchibstedGrotesk-Italic[wght].ttf`
- `SecularOne-Regular.ttf`

(Get them from Google Fonts.)

Then run:

```bash
npx react-native-asset
```

This auto-registers fonts in:
- iOS `Info.plist`
- Android `android/app/src/main/assets/fonts/`

---

## 4. Vector icons setup

### iOS
Add to `ios/WatchPartyLive/Info.plist` inside `<dict>`:

```xml
<key>UIAppFonts</key>
<array>
  <string>Ionicons.ttf</string>
  <string>FontAwesome.ttf</string>
</array>
```

### Android
Add to `android/app/build.gradle` (top of file):

```gradle
project.ext.vectoricons = [
    iconFontNames: [ 'Ionicons.ttf', 'FontAwesome.ttf' ]
]
apply from: "../../node_modules/react-native-vector-icons/fonts.gradle"
```

## 10. Troubleshooting

**Build fails on iOS with "Sandbox: rsync deny"**
Xcode 26 requires script sandboxing off. In Xcode → Project → Build Settings → User Script Sandboxing → No.

**`PhaseScriptExecution` fails with "Permission denied" on Hermes**
```bash
cd ios && pod deintegrate && pod install
```

**Android build fails: "Could not find tools.jar"**
Make sure `JAVA_HOME` points at JDK 17, not 8 or 11.

**Metro can't resolve `@/...` imports**
Babel cache: `npm start -- --reset-cache`.

**Vector icons show as `?` boxes**
You forgot the `Info.plist` / `build.gradle` step in section 4.

**Linear gradient: "RNCSafeAreaProvider not found"**
Reinstall pods: `cd ios && pod install`.
